# Colcon Install Layout — What Gets Generated

Traced from a real `colcon build --symlink-install` run.

## Per-package install layout (isolated mode)

```
install/<pkg>/
├── .catkin                         # empty marker (ament_cmake legacy)
├── include/<pkg>/                  # headers (if library)
├── lib/lib<pkg>.a                  # library (if library)
├── share/
│   ├── ament_index/resource_index/
│   │   ├── packages/<pkg>          # empty ament index marker
│   │   ├── parent_prefix_path/<pkg> # parent prefix (e.g. "/opt/ros/humble")
│   │   └── package_run_dependencies/<pkg>  # empty (run deps)
│   ├── colcon-core/packages/<pkg>  # colon-separated runtime dep names
│   └── <pkg>/
│       ├── cmake/                  # CMake config files (generated by ament)
│       │   ├── <pkg>Config.cmake
│       │   └── <pkg>Config-version.cmake
│       ├── environment/            # ament env hooks (from local_setup.dsv)
│       │   ├── ament_prefix_path.{dsv,sh}
│       │   └── path.{dsv,sh}
│       ├── hook/                   # colcon env hooks (from package.dsv)
│       │   ├── cmake_prefix_path.{dsv,sh}
│       │   ├── pkg_config_path.{dsv,sh}
│       │   ├── pkg_config_path_multiarch.{dsv,sh}
│       │   └── ros_package_path.{dsv,sh}
│       ├── local_setup.{bash,sh,zsh,dsv}
│       ├── package.{bash,sh,zsh,dsv}
│       └── package.xml
```

## Two layers of env hooks

### Layer 1: `environment/` — ament hooks (in `local_setup.dsv`)

Generated by ament_cmake during the cmake install step. These set up
the ament environment:

| Hook                    | Variable            | Suffix                                                     |
|-------------------------|---------------------|------------------------------------------------------------|
| `ament_prefix_path.dsv` | `AMENT_PREFIX_PATH` | (empty = prefix itself)                                    |
| `path.dsv`              | `PATH`              | `bin`                                                      |
| `library_path.dsv`      | `LD_LIBRARY_PATH`   | `lib` (only if lib/ has .so files)                         |
| `pythonpath.dsv`        | `PYTHONPATH`        | `local/lib/python3.X/dist-packages` (only Python packages) |

**rosup generates these.** ✅

### Layer 2: `hook/` — colcon hooks (in `package.dsv`)

Generated by colcon AFTER cmake install. These set up colcon-specific
paths:

| Hook                            | Variable            | Suffix                           |
|---------------------------------|---------------------|----------------------------------|
| `cmake_prefix_path.dsv`         | `CMAKE_PREFIX_PATH` | (empty = prefix itself)          |
| `pkg_config_path.dsv`           | `PKG_CONFIG_PATH`   | `lib/pkgconfig`                  |
| `pkg_config_path_multiarch.dsv` | `PKG_CONFIG_PATH`   | `lib/x86_64-linux-gnu/pkgconfig` |
| `ros_package_path.dsv`          | `ROS_PACKAGE_PATH`  | `share`                          |

**rosup does NOT generate these.** ❌ — This is the gap.

## Top-level install layout

```
install/
├── .colcon_install_layout          # "isolated"
├── COLCON_IGNORE                   # prevent colcon from scanning install/
├── _local_setup_util_sh.py         # Python helper for shell script generation
├── _local_setup_util_ps1.py        # PowerShell helper
├── local_setup.{bash,sh,zsh}      # source all per-package local_setup files
├── setup.{bash,sh,zsh}            # source chained prefixes + local_setup
```

The top-level `setup.bash` does two things:
1. Sources chained prefix paths (underlays like `/opt/ros/humble`)
2. Sources `local_setup.bash` for this install

The `local_setup.bash` reads `_local_setup_util_sh.py` to discover
all packages and source them in topological order.

**rosup does NOT generate the top-level scripts.** ❌ — colcon's dep
layer build generates these as part of its own install process.

## What rosup needs to add

### For the dep layer (`.rosup/install/`)

The dep layer is built by colcon, which generates all files automatically.
No rosup changes needed. ✅

### For the native build engine (Phase 11)

When rosup builds packages natively (without colcon), it needs to generate:

1. **`hook/` directory** — `cmake_prefix_path`, `pkg_config_path`,
   `pkg_config_path_multiarch`, `ros_package_path` DSV + shell files.
2. **`package.dsv`** — lists hooks from `hook/` directory.
3. **Top-level install scripts** — `setup.bash`, `local_setup.bash`,
   `_local_setup_util_sh.py` (or a simpler alternative).
4. **Additional ament index entries** — `parent_prefix_path`,
   `package_run_dependencies`.
5. **`.colcon_install_layout`** — marker file containing `"isolated"`.
6. **`.catkin`** — empty marker in each package prefix.

### Priority

The `hook/` directory hooks are critical — without `CMAKE_PREFIX_PATH`
being set, downstream packages can't find upstream cmake configs. The
current native builder works for isolated builds but would fail for
chained workspace builds.

### Current state

When rosup delegates to colcon (current `rosup build`), colcon generates
all files automatically — no gap. The gap only matters for the native
build engine (Phase 11).

For the native builder, the most important hooks to generate are:
1. `hook/cmake_prefix_path.{dsv,sh}` — essential for downstream cmake
2. `hook/pkg_config_path.{dsv,sh}` — for pkg-config consumers
3. `colcon-core/packages/<name>` — for topological sourcing
4. Top-level `setup.bash` — for users to source the workspace

The `environment/` hooks (ament_prefix_path, path, library_path) are
already generated by ament_cmake during cmake install. rosup's native
builder only needs to add the colcon-layer hooks.

### Who generates what

| File                                          | Generator                   | rosup native needs to add? |
|-----------------------------------------------|-----------------------------|----------------------------|
| `environment/ament_prefix_path.{dsv,sh}`      | ament_cmake (cmake install) | No                         |
| `environment/path.{dsv,sh}`                   | ament_cmake (cmake install) | No                         |
| `environment/library_path.{dsv,sh}`           | ament_cmake (cmake install) | No                         |
| `local_setup.{bash,sh,zsh,dsv}`               | ament_cmake (cmake install) | No                         |
| `package.{bash,sh,zsh,dsv,xml}`               | ament_cmake (cmake install) | No                         |
| `cmake/*Config.cmake`                         | ament_cmake (cmake install) | No                         |
| `ament_index/packages/<name>`                 | ament_cmake (cmake install) | No                         |
| `ament_index/parent_prefix_path/<name>`       | ament_cmake (cmake install) | No                         |
| `ament_index/package_run_dependencies/<name>` | ament_cmake (cmake install) | No                         |
| `hook/cmake_prefix_path.{dsv,sh}`             | **colcon post-install**     | **Yes**                    |
| `hook/pkg_config_path.{dsv,sh}`               | **colcon post-install**     | **Yes**                    |
| `hook/pkg_config_path_multiarch.{dsv,sh}`     | **colcon post-install**     | **Yes**                    |
| `hook/ros_package_path.{dsv,sh}`              | **colcon post-install**     | **Maybe** (catkin only)    |
| `colcon-core/packages/<name>`                 | **colcon post-install**     | **Yes**                    |
| `.catkin`                                     | **colcon post-install**     | **Yes** (trivial)          |
| Top-level `setup.bash`                        | **colcon post-install**     | **Yes**                    |
| `.colcon_install_layout`                      | **colcon post-install**     | **Yes** (trivial)          |
